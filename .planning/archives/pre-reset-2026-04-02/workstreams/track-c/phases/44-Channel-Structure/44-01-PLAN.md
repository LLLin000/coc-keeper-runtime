    Create new channel_enforcer.py module with:

    1. **ChannelType enum** - Defines channel roles:

       - ADMIN: 管理员频道
       - TRACE: KP-trace 频道
       - GENERAL: 未绑定频道

    2. **ChannelPolicy dataclass** - Maps commands to allowed channel types:
       ```python
       @dataclass
       class ChannelPolicy:
           command_names: list[str]
           allowed_types: set[ChannelType]
           redirect_message: str
       ```

    3. **ChannelEnforcer class** - Core enforcement logic:
       - `__init__(self, session_store: SessionStore)` - receives session store reference
       - `channel_type_for(guild_id: str, channel_id: str) -> ChannelType` - detects current channel role
       - `check_command(command_name: str, guild_id: str, channel_id: str) -> tuple[bool, str]` - returns (allowed, message)
       - `register_policy(policy: ChannelPolicy)` - adds command-to-channel mapping

    4. **Default policies** - Pre-configured for existing commands:
       - Archive commands (/sheet, /profiles, /profile_detail, /start_builder, /builder_reply): allowed in ARCHIVE only
       - Admin commands (/admin_profiles): allowed in ADMIN only
       - Game commands (/take_turn, /ready, /load_adventure, /enter_scene, /end_scene, combat commands): allowed in GAME only

    The ChannelEnforcer should follow the existing patterns from SessionStore (Pydantic models, dump/load for any state).
  </action>
