# Plan 40-01: Runtime Control Contracts And Shared Service Boundary

## Goal

Create the foundational runtime control contracts for Track E by introducing a shared control service, typed state and action result models, and integration seams that wrap existing restart and smoke-check behavior without changing their operational truth.

## Exit Criteria

- Track E has one reusable runtime control service.
- Runtime health is represented as a typed aggregated state instead of ad hoc log parsing in multiple places.
- Existing `smoke-check` and `restart-system` remain valid and reusable.
- Later phases can build a CLI surface and local web panel without re-implementing process and health logic.
