# Requirements: Discord AI Keeper - Track D

**Defined:** 2026-03-28
**Core Value:** Owns perceived table experience - Keeper-style presentation, clue/history/panel readability, and player-facing immersion without redefining canonical runtime truth.

## vD.1.1 Requirements (Track D)

Milestone: D1 Keeper-Guided Archive Experience

### Private-First Builder Experience

- [ ] **PRIVATE-01**: Character creation defaults to a more private-feeling interaction mode instead of feeling like a public form in the archive channel
- [ ] **PRIVATE-02**: Archive channels clearly explain when a reply continues builder flow versus when it is only viewing/managing profiles
- [ ] **PRIVATE-03**: Builder prompts feel like a Keeper shaping a person, not a generic questionnaire

### Archive Presentation

- [ ] **PRESENT-01**: Player-facing archive details feel like an investigator board rather than a thin command dump
- [ ] **PRESENT-02**: Rich archive sections remain readable in Discord message constraints
- [ ] **PRESENT-03**: Long-lived archive details and campaign-local details are visually distinguishable

### Future UI Contracts

- [ ] **ACTIVITY-01**: Archive presentation is defined in reusable sections suitable for a future Discord Activity panel
- [ ] **ACTIVITY-02**: Presentation-layer changes do not redefine archive ownership or canonical state

---

## vD.1.2 Requirements (Track D)

Milestone: Session Boards And Keeper Scene Presentation

### Session Boards

- [ ] **BOARD-01**: Player-facing boards can summarize campaign/adventure/session identity, current scene, recent history, and clue/current-state slices in one readable surface
- [ ] **BOARD-02**: Scene framing and consequence summaries are presented as coherent Keeper-facing updates rather than isolated bot replies
- [ ] **BOARD-03**: Session boards remain readable within Discord constraints while also staying suitable for future Activity migration

### Rules Boards

- [ ] **RULE-01**: Presentation layer can surface newcomer-friendly minimum-rules summaries without replacing the canonical rule source
- [ ] **RULE-02**: Presentation layer can surface digestible combat, injury, and SAN flow boards derived from canonical Track A rules
- [ ] **RULE-03**: Different explanation density can be supported for new-player, standard, and veteran session modes
