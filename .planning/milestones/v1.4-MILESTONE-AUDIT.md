# Milestone Audit: v1.4 房间图驱动的剧本理解与运行时

## Goal

将剧本理解从顺序文本块迁移到房间图与地点驱动的运行时，并让 AI 能先抽取出可审阅的 room graph 草稿，再把《疯狂之馆》迁移到新的 location-first 结构上。

## Outcome

Milestone `v1.4` met its core intent:

- Runtime now supports explicit `locations`, `start_location_id`, and adjacency-driven location transitions.
- `疯狂之馆` now ships with room-graph data for the central hall and each branch hall.
- Natural message play now distinguishes portal observation from actual entry and supports explicit room returns.
- The codebase now contains a reviewable AI-first extraction draft pipeline for room graphs and trigger summaries.

## What Landed

- Room-graph schema in packaged adventures
- Location-aware runtime snapshots and state
- AI extraction draft models and prompt path
- `mad_mansion` migration to explicit location graph data
- Regression coverage for movement, observation, extraction, and location snapshots

## Remaining Gap

- Trigger-tree consequence propagation after rolls is still thinner than the long-term target. The milestone introduced reviewable trigger summaries and location-first runtime behavior, but not a fully realized trigger execution engine.

## Verdict

`v1.4` is complete as a milestone. The main residual gap is narrow enough to seed the next milestone rather than block closure.
