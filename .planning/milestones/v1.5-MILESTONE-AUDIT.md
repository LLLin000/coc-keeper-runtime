# Milestone Audit: v1.5 通用 Trigger Tree 与后果引擎

## Goal

为 room-graph 模组补上通用、可链式执行的 trigger/consequence 引擎，让动作与检定结果能落成结构化后果、事件日志和可复用的模块推进，而不是只停留在 prompt 文字层。

## Outcome

Milestone `v1.5` met its core intent:

- The formal adventure schema now includes reusable trigger definitions with declarative conditions and effects.
- Runtime execution can resolve action and roll events into consequence chains, including state changes, clue reveals, pending-roll handoff, and location-aware summaries.
- Trigger execution now emits auditable event records instead of silently mutating state.
- `疯狂之馆` migrated key investigation and reveal beats onto the generic engine, proving the design can drive real module progress rather than only synthetic examples.

## What Landed

- Generic trigger node, condition, and effect models in adventure packages
- A reusable `TriggerEngine` for action, roll, and chained trigger execution
- Runtime integration for pending-roll consequence resolution and trigger event capture
- Extraction drafts extended with trigger-draft output for future AI-assisted module ingestion
- `mad_mansion` trigger migration for clock inspection, blood-hall wall reading, and life-hall research outcomes
- Diagnostics and Discord command flow now surface trigger-backed consequences more clearly

## Remaining Gap

- GM/operator control surfaces are still thinner than the long-term target. Event data exists, but richer trace routing, GM-facing views, and channel-layer controls remain the next natural follow-up.

## Verdict

`v1.5` is complete as a milestone. The core trigger/consequence runtime is in place, and the remaining work is product-surface polish on top of the new execution layer rather than missing engine fundamentals.
